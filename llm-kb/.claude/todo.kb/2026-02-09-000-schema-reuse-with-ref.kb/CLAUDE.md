# Schema-reuse effort history -- maintenance guide

Dated progress entries, decisions, and experiment findings from the
schema-reuse-with-$ref effort (`../2026-02-09-000-schema-reuse-with-ref.md`,
which rolls up this collection at a high level).

## What belongs here

One dated entry (`YYYY-MM-DD-NNN-slug.md`, chronological) per session's
progress, decision, or finding on this effort. Prose-only -- no
frontmatter, no schema; the date and kind live in the filename.

## What does NOT belong

- New work items -> `../` (todo.kb) or `.claude/ideas.kb/`
- Durable pattern documentation -> `references/schema-reuse.md`
- Migration state -> `migrations.kb/`
- Open-task bullets: never use `- [ ]` here -- these files sit under
  `todo.kb/` and the open-task inventory scans them for that marker;
  entries are history, not backlog.
