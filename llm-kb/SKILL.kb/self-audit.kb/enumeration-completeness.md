# Self-audit: enumeration completeness

## Goal

Every item the collection should hold exists as its own file, not
half-described inside another.

## Procedure

Re-run `../procedures.kb/enumerate-and-categorize.md` **afresh** --
list every item the collection should hold, categorized by type. Do
not rely on the original plan; the audit's value comes from
re-enumerating with current context.

Then for each enumerated item: does a file exist? Is anything
half-described in prose that should be promoted?

If the re-enumeration differs substantially from what's on disk, the
original plan was incomplete -- which is the failure this audit
catches.

## Recovery

Create missing files (`TODO: expand` stub is fine). Promote
half-described sections per
`../procedures.kb/promote-to-collection.md`.
