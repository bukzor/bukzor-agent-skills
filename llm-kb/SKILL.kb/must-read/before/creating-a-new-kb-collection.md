# Before Creating a New kb Collection

You're about to `mkdir $NAME.kb/` or otherwise stand up a new collection.

**Stop and read `references/creating-a-new-kb.md` first.**

That document contains the enumerate-and-categorize plan plus the four
quality passes (enumeration completeness, per-file scope, schema
potential, validation). Skipping leads to under-scoped collections that
need restructuring later -- a cost much larger than the read.

This applies whether you're creating the first `.kb/` in a project or
adding a sibling collection to an existing one.
