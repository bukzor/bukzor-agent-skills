# Procedures Are Tools

Every mechanical multi-step procedure is an engine command, not a
prose document. Prose procedures are permitted only where the steps
require judgment an agent must exercise (and even then, the
mechanical substeps call tools).

Checkable: for each prose procedure in the system, either it contains
a judgment step no tool could make, or it is a defect. The v1
promote-to-collection procedure — mechanical rename/split/rewire an
agent demonstrably failed to internalize — is the motivating
counter-example.
