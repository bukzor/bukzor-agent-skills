# Session Continuity

Many parallel sessions — across repos, days, and agents — stay
aligned, and no identified work is silently lost. Any agent dropping
into any session can recover: what is known, what was decided most
recently, what is in flight, and what it should do next.

Concretely: temporal ordering resolves conflicting knowledge (newest
wins); work items are machine-enumerable so backlog sweeps see
everything; session notes survive the session that wrote them.
