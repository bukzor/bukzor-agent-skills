# Synthesis File

`$name.md`, sibling of `$name.kb/`: the collection's roll-up.
`$name.kb/` elaborates; `$name.md` synthesizes.

Canonical uses:

- An open question: the `.md` states the question; entries are
  candidate resolutions; sub-questions recurse the same shape.
- A settled question: the `.md` carries the resolved answer; the
  collection keeps the alternatives-considered story off the hot
  path.
- A large organic document split into parts: the `.md` keeps the
  through-line.

A synthesis that merely enumerates contents is a defect (`ls` is the
index).

The relation is specialized by each application of the spec to a
domain: a task collection's synthesis is its live working set (one
line captures a tangent without taking it); a question's is its
answer; free-form is allowed. One tree commonly mixes several
notions of synthesis at different spots.
