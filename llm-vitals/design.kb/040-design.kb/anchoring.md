# Anchoring (Habit Stacking)

Named after BJ Fogg's Tiny Habits and James Clear's habit stacking:
new behaviors are reliably installed by tying them to an existing
trigger that already fires reliably.

The system's two anchors:

- **session-start** (existing skill) → picker output displays
- **session-end** (existing skill) → day-log entry prompts

Both anchors are already part of the operator's Claude Code
workflow. The vitals system adds output to surfaces that already
exist; it does not create a new ritual.

Rejected alternatives:

- **Daily calendar reminder**: fails because calendar reminders
  reliably decay.
- **Standalone dashboard tab**: fails because it requires remembering
  to open it.
- **Push notification on a clock**: produces alert fatigue; people
  dismiss recurring notifications.

The principle: anchor new behaviors to existing fired triggers, never
to memory or calendar.
