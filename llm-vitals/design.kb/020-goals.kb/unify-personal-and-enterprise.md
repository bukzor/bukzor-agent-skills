# Unify Personal and Enterprise Domains

A one-person enterprise has no clean boundary between personal and
business concerns. The operator's body, relationships, restoration,
revenue, customers, ops, marketing, and monitoring are all domains
that need minimum maintenance to avoid rot.

The system treats them as instances of a single primitive (the
vital), differing only in configuration and which private repo holds
the data. Adding a wellness vital is the same architectural operation
as adding an ops vital — both are configuration, not code changes.

This unification is structurally significant: if wellness and
enterprise extensions are mechanically identical, the design has been
factored correctly.
