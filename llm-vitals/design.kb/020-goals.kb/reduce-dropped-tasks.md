# Reduce Dropped Tasks

The operator records many intentions across task systems, kbs, and
session notes. Volume exceeds working memory; items push past
attention. Some recorded items are quietly important and get
forgotten.

The system reduces dropped balls by:

1. Surfacing items by current debt rather than by raw recency
2. Compressing many lists into one debt-ranked view
3. Making the most-in-debt domain visible at every session start

Success is measured by: previously-buried items resurfacing without
prompted search; reduction in "I forgot about X for months" moments
that turn out to be costly.
