# Enable Accountable Deep Work

The operator has a productive bent toward rabbit-hole exploration —
sometimes a multi-domain leverage win, sometimes a runaway. The
system's job is not to suppress rabbit holes but to make them
**accountable**: stated thesis up front, declared burn budget, exit
signal, and post-hoc scoring against the original claim.

Over time, this produces a personal calibration record — the operator
learns which categories of deep work pay off and which don't — which
is the only honest signal for distinguishing productive from
unproductive rabbit holes.

Deferred to later phases (see `070-future-work.kb/`); listed here as a
goal so the design accommodates it from the start rather than treating
it as bolt-on.
