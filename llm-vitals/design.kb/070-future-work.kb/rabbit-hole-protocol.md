# Rabbit-Hole Protocol

Pre-entry gate, mid-hole intercept, and post-hole postmortem for
deep-work exploration sessions.

**Pre-entry gate** — three required lines:

- *Claimed beneficiaries* — which vitals/themes this could help
- *Burn budget* — max hours before forced reassessment
- *Exit signal* — what would tell me to abandon early

**Mid-hole intercept** — at burn-budget exhaustion, the system
surfaces (doesn't terminate). Options: commit another tranche,
exit with findings, exit and discard.

**Post-hole postmortem** — one line per claimed beneficiary: did it
benefit? Yes/no.

Over time this builds a personal calibration record: hit-rate per
claim category, accuracy of burn budgets, which themes genuinely
cross-leverage vs. which are wishful.

Deferred because: needs accumulated postmortem data to be useful.
Graduates when the operator has ~20 logged sessions and can derive
a non-trivial baseline.
