# Support Both Vital Kinds

The system must accommodate two reporting mechanisms uniformly:

- **Journal-kind**: a `.kb/` of dated entries (suited to wellness, where
  the unit is reflective text and the texture cannot be hours-counted)
- **Task-kind**: frontmatter tag on existing task items (suited to
  work, where the existing task infrastructure already captures
  activity)

Both must fit the same tier-1/tier-2 model and the same picker
surface. The choice of kind is per-vital configuration; the
architecture does not privilege one over the other.

This is what makes wellness and enterprise extensions architecturally
identical: only the reporting mechanism differs.
