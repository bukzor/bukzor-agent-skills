# Visible Debt

For each vital, the system must expose a single visible number
quantifying current debt — typically "time since the floor was last
met" or "magnitude of current SLO violation."

Visibility must be **default**, not opt-in: every session-start view
shows the largest current debts without the operator having to ask.

Visible debt is the primary forcing function: it makes imbalance
impossible to ignore without being coercive. It defends against
multiple failure modes (forgotten items, biased attention, invisible
wellness slippage) with the same mechanism.
